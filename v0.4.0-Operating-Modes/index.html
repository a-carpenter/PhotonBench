Compare SNR curves (up to 5). The left plot shows raw SNR; the right plot normalizes SNR to a 13 µm reference pixel. Correcting for pixel surface area allows cameras with different pixel sizes can be compared fairly when within the same optical system.

This panel renders a live, false-color simulated sensor frame using the current Experimental and Camera Parameters, with shot, dark, and read noise applied per pixel. A circular spot of illumination is placed at the sensor's center; pixels along its edge are anti-aliased, receiving a fractional share of the illumination rather than a hard on/off cutoff.

This view is for visualization only. While Play is active, the frame redraws with fresh random noise about 5 times per second. This refresh rate is a display setting, not a simulated camera frame rate, and has no effect on exposure time or any other part of the simulation.

This histogram shows the distribution of ADU values across every pixel in the current simulated frame, binned into 80 bins spanning the sensor's readout range. Two populations are typically visible: a peak near the background level and a peak near the illuminated signal level, each widened by noise. Pixels left unilluminated by a sensor size that isn't an exact multiple of the active bin factor are excluded. The toggle in the header switches the y-axis between linear and log scale — log makes small populations near the tails easier to see, while linear avoids exaggerating them.

This plot shows a single row of ADU values across the sensor, taken from the middle row of the current frame — the same row marked by the colored line overlaid on the Camera Simulator panel. It's a horizontal slice through the illuminated spot, showing the signal rise through the anti-aliased edge, plateau across the fully illuminated pixels, and fall back to background, with pixel-to-pixel noise visible throughout.

Displayed is the analytic Signal-to-Noise Ratio as a function of the number of incident photons of a pixel. Computed directly from the current Experimental and Camera Parameters rather than from the simulated frame. The shaded band shows fluctuations about the SNR curve resulting from variation in the combinations of noise and the red marker tracks your current Photons setting.

When EM Gain and/or Binning are active, the baseline "Single Pixel SNR" curve becomes dashed and a solid "Modified SNR" curve is added showing the actual current-settings performance.

Use Compare to save the active curve for side-by-side comparison in the Camera Sensitivity Comparison panel.

Adjustable experimental and camera parameters to simulate the camera performance of your choice.

Camera Type

Choose from CCD, sCMOS, or InGaAs. This will determine how pixels are binned and will adjust the camera properties. Defaults are inspired by CCD, sCMOS, and InGaAs array detectors found in the scientific camera market.

Experimental Parameters

Photons (#)
Adjusts the number of photons/pixel in the illuminated area.
Exposure Time (s)
Interacts with the dark current to model thermal noise to the simulation. Does not interact with photons or any other part of the simulation.
Spot Radius (px)
Pixel radius of illuminated circle in the simulation.

Camera Parameters

Quantum Efficiency
Manufacturer specification for average probability of a photon being detected. Set for the relevant wavelength and camera sensor you are interested in.
Dark Current (e-/px/s)
Manufacturer specification for average thermal noise performance of a camera's pixels.
Read Noise (e-)
Manufacturer specification for noise introduced at readout.
Full Well Depth (e-)
Manufacturer specification for how many photoelectrons can be held by an individual pixel.
Register Well Depth (e-)
Specific to CCDs. Manufacturer specification for how many photoelectrons can be held by an individual pixel in a CCD's read register.
Offset (e-)
Uncommon to see reported in manufacturer specification sheet, but present in most modern camera technology to prevent negative values. Sometimes called a DC offset. Included in the model for completeness.
Sensitivity (e-/ADU)
Uncommon to see reported in a manufacturer specification sheet but dictates how many photoelectrons equals a count. Included in the model for completeness.
Pixel Size (µm)
Size of the camera sensor's pixel. Used for SNR curve normalization.
Bit Depth
Manufacturer specification for the number of "grey levels" after a signal has been digitized.
Binning
Default is unselected. Available binning options include 1, 2, 4, 8 in the vertical and horizontal directions. Signal and noise summation is defined by the chosen camera type, and the simulation panel slightly changes the size so there are no dead columns or rows of pixels.
Enable EM Gain
Specific to CCD. Models the effects of electron multiplying CCDs. Included in the model for completeness. For simplicity the EM gain register well depth is assumed to be the same as the Register Well Depth. This is not guaranteed and is often specified in manufacturer specification sheets.

The simulated spectrum is calculated by vertically binning the sensor's illuminated rows into a single row of ADU values, using the same signal and noise model as the Camera Simulator panel. The same binning rules for your selected camera type (CCD, sCMOS, InGaAs) apply here as everywhere else in the simulation. When Full Sensor Vertical Bin is checked, every row on the sensor contributes; when unchecked, only the rows inside the Custom ROI (Top to Bottom) are combined. Horizontal Binning, if active, further combines neighboring columns, reducing the number of points along the x-axis and improving each point's SNR.

The toggle in the header switches the x-axis between Pixel (native column index) and Wavelength (nm), using a generic Czerny-Turner grating dispersion model driven by the Dispersion Model parameters in the panel below. Wavelength is always computed from each point's native pixel center, so Horizontal Binning changes the point spacing but not the underlying wavelength mapping. If the current Dispersion Model settings describe a grating geometry with no physical solution (e.g. too high a groove density for the chosen center wavelength and angle), the chart falls back to the Pixel axis until the parameters are adjusted.

ROI & Binning — Region of Interest settings including Full Sensor Vertical Bin, Custom ROI settings, and Horizontal Binning (linked to primary parameter panel).

Dispersion Model — Input parameters for a generic Czerny-Turner dispersion model used to map the pixel axis onto a wavelength axis.

SNR vs. ROI Height — SNR as a function of the ROI height to illustrate how SNR can scale for flat field illuminated vertical bin. ROI height is linked to the image sensor height used in the simulation.
